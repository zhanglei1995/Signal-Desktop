## 简介
Signal-Desktop的语言选择策略是一个多层次的系统，它综合考虑用户操作系统区域设置、应用配置文件设置和用户手动选择等多种因素来确定最终显示语言。该系统设计旨在提供最佳的用户体验，确保用户能够以最熟悉的语言与应用交互。

## 语言选择优先级
Signal-Desktop的语言选择遵循一个明确的优先级顺序：

1. **用户手动选择**：用户在应用设置中明确选择的语言具有最高优先级
2. **操作系统区域设置**：如果用户未手动选择语言，则使用操作系统的首选语言设置
3. **默认语言**：如果前两者都无法确定，则使用英语作为默认语言

在`app/locale.node.ts`文件中，`load`函数实现了这一优先级逻辑：
```typescript
const matchedLocale = LocaleMatcher.match(
  localeOverride != null ? [localeOverride] : preferredSystemLocales,
  availableLocales,
  'en',
  { algorithm: 'best fit' }
);
```

这个匹配过程使用了`@formatjs/intl-localematcher`库，它遵循ICU（International Components for Unicode）的区域设置匹配规则，能够智能地匹配最合适的语言变体。

## 配置文件存储格式
语言设置存储在用户数据目录的`config.json`文件中，其结构如下：

```json
{
  "localeOverride": "zh-CN"
}
```

`localeOverride`字段存储用户手动选择的语言代码。如果该字段不存在或为null，则表示使用系统语言。

在`app/user_config.main.ts`文件中，系统初始化用户配置：
```typescript
export const userConfig = start({
  name: 'user',
  targetPath,
  throwOnFilesystemErrors: true,
});
```

这个配置系统使用Electron的`app.getPath('userData')`来确定用户数据目录的位置，确保配置文件在不同操作系统上都能正确存储。

## 用户界面实现
语言选择功能在用户界面中通过"外观"设置页面提供。在`ts/components/Preferences.dom.tsx`文件中，实现了语言选择的UI组件：

```typescript
const availableLocalesOptions = availableLocales
  .map(locale => {
    const currentLocaleLabel = getLocaleDisplayName(resolvedLocale, locale);
    const matchingLocaleLabel = getLocaleDisplayName(locale, locale);
    return { locale, currentLocaleLabel, matchingLocaleLabel };
  })
  .sort((a, b) => {
    return collator.compare(a.locale, b.locale);
  });
```

用户界面包含以下特性：
- 按字母顺序排序的语言列表
- 搜索功能，允许用户快速查找特定语言
- 显示语言的本地化名称和匹配名称
- 确认对话框，提示用户更改语言后需要重启应用

## 回退机制
Signal-Desktop实现了完善的回退机制，确保在特定语言包不完整时能够优雅降级：

1. **消息回退**：当特定语言的消息不完整时，系统会回退到英语版本
```typescript
// 我们从英语开始，然后用本地语言中存在的内容覆盖
finalMessages = merge(englishMessages, matchedLocaleMessages);
```

2. **区域设置规范化**：使用`resolveCanonicalLocales`函数确保区域设置代码的规范性
```typescript
export function resolveCanonicalLocales(locales: Array<string>): Array<string> {
  const validLocales = locales.filter(locale => {
    try {
      new Intl.Locale(locale).maximize();
      return true;
    } catch {
      return false;
    }
  });
  const canonical = Intl.getCanonicalLocales(validLocales);
  if (canonical.length === 0) {
    return ['en'];
  }
  return canonical;
}
```

3. **文本方向检测**：自动检测语言的文本方向（从左到右或从右到左）
```typescript
function getLocaleDirection(
  localeName: string,
  logger: LoggerType
): LocaleDirection {
  const locale = new Intl.Locale(localeName);
  try {
    if (typeof locale.getTextInfo === 'function') {
      return parseUnknown(
        TextInfoSchema,
        locale.getTextInfo() as unknown
      ).direction;
    }
    if (typeof locale.textInfo === 'object') {
      return parseUnknown(
        TextInfoSchema,
        locale.textInfo as unknown
      ).direction;
    }
  } catch (error) {
    logger.error(
      'locale: Error getting text info for locale',
      Errors.toLogFormat(error)
    );
  }
  return 'ltr';
}
```

## 实际案例分析

### 首次安装场景
当用户首次安装Signal-Desktop时：
1. 系统读取操作系统的首选语言设置
2. 使用`LocaleMatcher.match`找到最匹配的可用语言
3. 如果匹配成功，使用该语言；否则使用英语
4. 在配置文件中不设置`localeOverride`，表示使用系统语言

### 设置变更场景
当用户在设置中更改语言时：
1. 用户在"外观"设置中选择新语言
2. 系统将选择的语言代码存储到`config.json`中的`localeOverride`字段
3. 应用提示用户需要重启以应用更改
4. 重启后，`localeOverride`的优先级高于系统设置

### 多用户环境场景
在多用户环境中，每个用户的语言设置是独立的：
1. 每个用户有自己的`userData`目录
2. 语言设置存储在各自的`config.json`文件中
3. 系统根据当前登录用户加载相应的语言设置
4. 不同用户可以使用不同的界面语言

这些案例展示了Signal-Desktop语言选择系统的灵活性和用户友好性，确保在各种使用场景下都能提供最佳的语言体验。
