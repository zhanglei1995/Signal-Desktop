## 简介
Signal-Desktop使用Redux进行全局状态管理，其中语言相关的状态存储在用户状态slice中。本文档详细说明了语言状态的管理机制，包括`userChanged` action如何更新Redux store中的语言相关属性，用户状态slice的结构，状态更新的不可变性原则，以及语言偏好在应用重启后如何持久化。

## 语言变更操作
`userChanged` action是更新用户状态的主要方式。当用户更改语言设置时，会触发此action来更新Redux store中的相关属性。

```mermaid
sequenceDiagram
participant UI as 用户界面
participant Action as Action Creator
participant Reducer as Reducer
participant Store as Redux Store
UI->>Action : 调用userChanged(attributes)
Action->>Action : 创建USER_CHANGED action
Action->>Store : dispatch(action)
Store->>Reducer : 调用reducer函数
Reducer->>Reducer : 检查action.type
Reducer->>Reducer : 返回新状态
Reducer-->>Store : 更新store
Store-->>UI : 通知状态变更
```

## 状态更新的不可变性
在Redux中，状态更新必须遵循不可变性原则。Signal-Desktop通过使用扩展运算符(`...`)来确保状态的不可变性。

```mermaid
flowchart TD
Start([开始]) --> CheckAction["检查action.type是否为'USER_CHANGED'"]
CheckAction --> |是| ExtractPayload["提取payload数据"]
ExtractPayload --> CreateNewState["创建新状态对象"]
CreateNewState --> ReturnNewState["返回新状态"]
CheckAction --> |否| ReturnOriginal["返回原始状态"]
ReturnOriginal --> End([结束])
ReturnNewState --> End
```

## 状态持久化机制
语言偏好设置在应用重启后需要保持不变，这通过状态持久化机制实现。Signal-Desktop使用localeOverride来存储用户的语言选择。

```mermaid
sequenceDiagram
participant App as 应用程序
participant Locale as 语言模块
participant Config as 配置系统
App->>Locale : 启动时加载语言设置
Locale->>Config : 获取localeOverride
Config-->>Locale : 返回存储的语言偏好
Locale->>Locale : 匹配最合适的语言
Locale-->>App : 初始化i18n系统
App->>App : 显示相应语言的界面
```

## 语言选择界面
用户通过偏好设置界面选择语言。选择后需要重启应用以应用新的语言设置。


## 总结
Signal-Desktop的语言状态管理通过Redux实现，使用`userChanged` action来更新用户状态slice中的语言相关属性。状态更新遵循不可变性原则，通过扩展运算符创建新状态对象。语言偏好通过localeOverride机制持久化存储，确保在应用重启后保持用户的语言选择。整个系统设计确保了语言切换的可靠性和一致性。
