## 简介

Signal-Desktop的捐赠功能为用户提供了一种安全、便捷的一次性捐赠方式。该功能通过一系列精心设计的React组件和后端服务实现，确保捐赠流程的顺畅和安全。核心组件包括金额输入框、信用卡信息输入框（卡号、有效期、CVC码）以及支付网关的集成接口。整个流程从用户选择捐赠金额开始，经过信用卡信息输入和验证，最终完成支付并展示结果。系统还考虑了无障碍访问、响应式布局和安全合规性（如PCI DSS）的要求。

## 金额输入与验证

### DonateInputAmount 组件

`DonateInputAmount` 组件负责处理用户输入的捐赠金额。该组件支持多种货币格式，并根据所选货币自动应用相应的格式化规则。组件使用 `useInputMask` 钩子来实现输入掩码，确保用户输入的金额符合预期格式。

#### 输入格式化

- **货币符号**：根据所选货币自动添加前缀或后缀符号。
- **数字分组**：使用千位分隔符（如逗号）分隔大额数字。
- **小数点**：根据货币的精度要求，限制小数位数（最多2位）。
- **零值处理**：防止用户输入多个前导零。

#### 验证规则

- **最小金额**：确保输入的金额不低于该货币的最低捐赠金额。
- **最大金额**：确保输入的金额不超过Stripe支持的最大金额。
- **有效数字**：仅允许输入数字、小数点和千位分隔符。
- **小数位数**：对于零小数位货币（如日元），不允许输入小数部分。

#### 用户体验优化

- **光标位置**：当用户点击输入框时，光标会自动定位到正确的位置，避免因货币符号导致的光标错位。
- **占位符**：提供清晰的占位符文本，指导用户输入金额。
- **实时反馈**：在用户输入过程中，实时显示格式化后的金额，提高输入准确性。

## 信用卡信息输入组件

### DonateInputCardNumber 组件

`DonateInputCardNumber` 组件用于输入信用卡卡号。该组件实现了以下功能：

- **输入掩码**：使用 `CC_NUMBER_FORMATTER` 格式化器，自动在卡号中插入空格，使其更易读（例如：`0000 0000 0000 0000`）。
- **输入模式**：设置 `inputMode="numeric"`，在移动设备上显示数字键盘。
- **自动补全**：设置 `autoComplete="cc-number"`，允许浏览器自动填充信用卡卡号。
- **最大长度**：限制输入的最大长度为19个字符（包括空格）。

#### 验证规则

- **空值检查**：确保卡号不为空。
- **字符验证**：仅允许输入数字和空格。
- **长度验证**：确保卡号长度符合标准信用卡卡号的长度要求。
- **有效性检查**：使用Luhn算法验证卡号的有效性。

### DonateInputCardExp 组件

`DonateInputCardExp` 组件用于输入信用卡的有效期。该组件实现了以下功能：

- **输入掩码**：使用 `CC_EXP_FORMATTER` 格式化器，自动在有效期中插入斜杠（例如：`MM/YY`）。
- **输入模式**：设置 `inputMode="numeric"`，在移动设备上显示数字键盘。
- **自动补全**：设置 `autoComplete="cc-exp"`，允许浏览器自动填充有效期。
- **占位符**：提供本地化的占位符文本，指导用户输入有效期。

#### 验证规则

- **空值检查**：确保有效期不为空。
- **字符验证**：仅允许输入数字和斜杠。
- **格式验证**：确保输入的格式为 `MM/YY`。
- **月份验证**：确保月份在1到12之间。
- **年份验证**：确保年份不早于当前年份，且不超过未来5年。
- **过期检查**：确保信用卡未过期。

`DonateInputCardCvc` 组件用于输入信用卡的安全码（CVC）。该组件实现了以下功能：

- **字符验证**：仅允许输入数字。
- **长度验证**：确保CVC码长度符合标准要求（3位或4位）。
- **有效性检查**：确保CVC码为有效数字。

## 表单数据绑定与错误处理

### 数据绑定

捐赠表单中的各个输入组件通过 `value` 和 `onValueChange` 属性与父组件进行数据绑定。父组件（如 `PreferencesDonateFlow`）维护一个状态对象，存储用户输入的所有信息。当用户在输入框中输入内容时，`onValueChange` 回调函数会被触发，更新父组件的状态。

### 错误处理

#### 实时验证

- **输入时验证**：在用户输入过程中，实时检查输入内容是否符合格式要求。
- **失焦时验证**：当用户离开输入框时，进行更严格的验证，确保输入内容完全正确。

#### 错误提示

- **内联错误消息**：在输入框下方显示具体的错误消息，帮助用户理解问题所在。
- **视觉反馈**：使用红色边框或图标突出显示有错误的输入框。
- **国际化**：错误消息通过 `i18n` 函数进行本地化，支持多种语言。

#### 错误类型

- **空值错误**：输入框为空。
- **格式错误**：输入内容不符合预期格式。
- **长度错误**：输入内容长度超出或不足。
- **逻辑错误**：输入内容在逻辑上不合理（如过期日期）。

## 支付网关集成

### 安全传输

- **HTTPS**：所有与支付网关的通信都通过HTTPS加密，确保数据传输的安全性。
- **Tokenization**：信用卡信息在客户端被转换为令牌，避免敏感信息在网络上传输。

### 令牌化处理

- **Stripe.js**：使用Stripe的JavaScript库处理信用卡信息，生成一次性令牌。
- **客户端处理**：信用卡信息在用户的浏览器中直接发送给Stripe，不会经过Signal的服务器。

### 支付状态回调

- **状态机**：使用状态机管理捐赠流程的不同阶段（如待确认、处理中、已完成）。
- **回调函数**：支付网关在不同阶段调用相应的回调函数，更新捐赠状态。
- **错误处理**：在支付失败时，提供详细的错误信息，并允许用户重新尝试。

#### 支付流程

1. **创建支付意图**：调用 `createPaymentIntent` API 创建支付意图。
2. **确认支付**：调用 `confirmPayment` API 确认支付。
3. **获取收据**：支付成功后，调用 `getReceipt` API 获取电子收据。
4. **完成捐赠**：更新用户界面，显示捐赠成功消息。

## 捐赠流程用户引导

### 用户引导设计

- **分步流程**：将捐赠流程分为多个步骤，每一步都有明确的指示。
- **进度反馈**：在用户进行每一步操作时，提供即时的反馈，如加载动画或成功提示。
- **结果展示**：捐赠成功后，显示感谢消息和电子收据。

### 进度反馈

- **加载状态**：在支付处理过程中，显示加载动画，告知用户正在处理。
- **成功状态**：支付成功后，显示绿色的对勾图标和成功消息。
- **失败状态**：支付失败时，显示红色的叉号图标和错误消息。

### 结果展示

- **感谢消息**：显示个性化的感谢消息，增强用户满意度。
- **电子收据**：提供电子收据的下载链接，方便用户保存。
- **分享选项**：允许用户通过社交媒体分享捐赠经历。

## 安全与合规性

### 无障碍访问

- **键盘导航**：确保所有功能都可以通过键盘操作。
- **屏幕阅读器支持**：为视障用户提供屏幕阅读器友好的标签和描述。
- **高对比度模式**：支持高对比度模式，方便色盲用户使用。

### 响应式布局

- **自适应设计**：确保捐赠表单在不同设备上都能正常显示。
- **触摸友好**：在移动设备上，输入框和按钮的大小适中，便于触摸操作。

### 安全合规性

- **PCI DSS**：遵循支付卡行业数据安全标准（PCI DSS），确保信用卡信息的安全。
- **数据最小化**：仅收集必要的信息，减少数据泄露的风险。
- **定期审计**：定期进行安全审计，确保系统的安全性。
